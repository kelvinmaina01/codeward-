1. Bridging the UI and the Backend

The Hono RPC & WebSocket Strategy

To eliminate mock data, we must bridge `apps/web` to `apps/api`. We will NOT build disjoint REST endpoints. We will use Hono RPC (hono/client) to achieve end-to-end type safety from the DB schema all the way to the React props.

The Data Bridge: Hono RPC

We will export the Hono `AppType` from `apps/api/src/index.ts` and use `hc` (Hono Client) in the frontend. This gives us autocomplete on all DB models (like `runs` and `agent_tasks`) inside React without manually defining types.

The Real-Time Bridge: WebSockets

We will introduce `@hono/node-ws`. The frontend will connect a WebSocket to `/ws/live-feed`. The backend will listen to BullMQ `completed` and `progress` events, broadcasting JSON to the frontend to animate the agent rings.

Database Schema Expansion

We will expand `db/schema.ts` to support Phase 2:

  • Add aiCredits integer column to the organization table to track billing.
  • Add visibility enum (private/public) to runs to support the Global Feed.

2. Connecting a New Repo (The Missing Onboarding)

GitHub App OAuth & Sync

A user must be able to securely connect their repository. We cannot run agents without code.

GitHub App Installation

We will build an endpoint /api/github/install that redirects the user to the GitHub App installation URL. Upon successful callback, we capture the installation_id and sync their accessible repositories to our repositories table.

ConnectRepo.tsx UI

We will implement the ConnectRepo.tsx screen. It will fetch the synced repos via Hono RPC, list them with a "Connect" button, and trigger the very first "Baseline Audit" run to score the repo.

3. AI Credits & Stripe Billing

The Cost of Intelligence

Agents are expensive. Haiku and Sonnet burn tokens parsing ASTs. We must monetize and restrict access to prevent abuse.

  • Credit Ledger: Add a credit_ledger table to log every deduction (e.g., -150 credits for a Sonnet-4.5 security scan). This maps directly to the tokenUsage in agent_tasks.
  • Stripe Integration: Build a /api/billing/checkout endpoint using Stripe Checkout. Users buy "Credit Packs" (e.g., $50 for 10,000 AI Credits).
  • Free Tier: New organizations get 1,000 free credits on sign up (enough for 2-3 baseline audits) to test the platform.
  • Out of Credits: The webhook receiver will check the organization.aiCredits balance. If it's ≤ 0, the webhook is rejected and the agents do not run.

4. The Codeward AI Command Center (`AIAgent.tsx`)

The AI Agent panel will evolve into a fully threaded chat experience powered by `claude-sonnet-4-5`.

  • Tool Use: The chat agent will have access to `query_run_history`, `explain_debt_item`, and `spawn_security_scan`.
  • Memory: We will implement the `agent_memory` database queries so the AI remembers context across sessions.
  • Streaming: Implement Server-Sent Events (SSE) so the Claude response streams directly into the side-pull drawer.

5. The Diff Viewer & GitHub Workflow

When the Bloat Agent refactors a function, we must render the diff and allow the user to approve it.

Component: DiffViewer

Build a split diff viewer. Show the old AST logic vs the AI's optimized logic. Render comments explaining the improvements.

Component: Staging/Approvals

Wire the "Approve and Merge" button to the GitHub App API (`github/client.ts`). Approving in the dashboard triggers a PR merge.

6. Gamification Engine

As requested, we will expand the gamification aspects to encourage engineers to clear their technical debt.

  • Global Feed: Set up a Supabase view for opted-in "cleared debt" events to stream achievements across all organizations.
  • The Leaderboard: Rank engineers based on their cumulative "Debt Removed" score. Render real badges (Gold, Silver, Bronze) in the UI.
  • Settings Opt-in: Build `Settings.tsx` for privacy toggles: "Show my cleared debt on the Global Feed".
